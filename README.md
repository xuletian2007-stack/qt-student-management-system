基于 Qt 的学生信息管理系统

## 一、项目简介

本项目是一个基于 Qt 开发的学生信息管理系统，主要用于完成学生信息的录入、查询、修改、删除、文件保存与读取等功能。

基础功能：
1. 记录课程成绩并进行GPA计算
2. 记录成就（奖项，班级代表，志愿者）
3. 导出一份完整的个人简历

创新功能：调用api让AI生成一个个人简历网页

非功能需求：

离线运行：必须作为本地可执行程序运行，无需连接服务器即可处理核心数据 

版本管理：使用 Git 进行开发过程管理 

风格选择：MVC (模型-视图-控制器) 风格

·Model (模型)：负责 GPA 计算逻辑和本地数据的存取（如课程信息、奖项记录） 。

·View (视图)：利用 Qt 框架开发本地窗口界面，展示学生信息及 AI 生成的简历网页 。

·Controller (控制器)：处理用户点击逻辑，并负责调用 AI API 进行跨模块通信。

优势：实现了业务逻辑与界面的解耦，方便 4-5 人的团队进行模块化分工协作 。



## 二、用例分析

三类参与者：

•	大学生 (Student)：核心用户，负责记录学习历程、管理成就并触发简历生成 。 

•	管理员 (Administrator)：负责从教务系统或外部文件批量导入课程和初始成绩 。 

•	AI API 服务 (External AI Service)：支撑创新功能的外部系统（如 Deepseek API），负责处理数据并返回 HTML 简历代码 。

Use Case 1：批量导入课程与成绩

主要参与者：管理员 (Administrator)

前置条件：管理员已准备好符合格式要求的外部数据文件（如 CSV 或 JSON）

基本路径：

1.	管理员在系统设置中选择“批量导入”功能
2.	系统弹出文件选择对话框，管理员选择本地数据文件
3.	系统校验文件格式及数据的合法性（如成绩是否在 0-100 之间）
4.	系统解析文件内容，将课程名称、学分、初始成绩写入本地数据库
5.	系统提示导入成功，并自动触发一次全局 GPA 更新
   
后置条件：系统本地数据库更新，学生可直接查看已录入的学习记录 

————————

Use Case 2：查看 GPA 统计分析

主要参与者：大学生 (Student)

前置条件：系统中已存在至少一门已评分的课程记录

基本路径：

1.	用户进入“学业看板”页面。
2.	系统自动提取所有课程数据，应用 GPA 计算公式 ： 
$$GPA = \frac{\sum (成绩点 \times 学分)}{\sum 总学分}$$
3.	系统将计算结果以数字和折线图（展示各学期趋势）的形式展示给用户
   
后置条件：用户直观掌握自己的学业水平

————————

Use Case 3：记录个人成就与角色

•	主要参与者：大学生 (Student)

•	前置条件：用户处于“成长历程记录”页面

•	基本路径：

1.	用户选择记录类别（奖项、班级角色、志愿者经历、其他）
2.	用户填写具体内容（标题、时间、级别、职责描述）
3.	用户上传相关证明材料（可选，如奖状照片路径）
4.	用户点击“保存”，系统将其按时间顺序排列并存储
   
后置条件：个人成就库更新，作为后续生成简历的原始素材

————————

Use Case 4：导出标准个人简历

•	主要参与者：大学生 (Student)

•	前置条件：系统内已有足够的个人数据

•	基本路径：

1.	用户点击“导出简历”按钮
2.	系统提取所有成绩、奖项及角色信息
3.	系统套用预设的简历模板，将零散数据整理成规范文档
4.	系统弹出保存对话框，用户选择保存路径（如桌面）
   
后置条件：在本地生成一份完整的简历文件

————————

Use Case 5：数据备份（导出）与恢复（导入）

•	主要参与者：大学生 (Student)

•	基本路径：

1.	用户选择“数据备份”，系统将当前所有记录打包成一个加密文件，方便迁移
2.	用户选择“数据恢复”，系统读取备份文件并覆盖/合并当前本地数据库
   
后置条件：确保学生四年的记录不会因重装系统或软件而丢失
